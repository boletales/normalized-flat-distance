# ドキュメント記録要件（エージェント参照用）

本ファイルは、開発中に「何をどこへ記録するか」を固定するための運用ルールである。
エージェントおよび開発者は、ドキュメント更新時にこの要件を参照すること。

## 記録先の責務

- 「開発の要件」＋「検討中の事柄・判断の根拠」
  - 記録先: `docs/design.md`
- 「エージェントの記録」＋「開発の履歴」
  - 記録先: `docs/development-log.md`
- 「コード概略」
  - 記録先: `docs/code-overview.md`（独立して保持）

## 更新ルール

1. 要件・仕様・設計根拠を変更した場合は、`docs/design.md` を先に更新する。
2. 実装作業や判断履歴を残す場合は、`docs/development-log.md` に日付付きで追記する。
3. モジュール構成や公開APIの説明を変更した場合は、`docs/code-overview.md` を更新する。
4. 複数ファイルにまたがる変更では、最後に整合性確認（記述の矛盾がないか）を行う。

## 目的

- 記録の重複・欠落を防ぐ
- エージェントの更新先判断を一貫させる
- レビュー時に変更意図を追跡しやすくする
