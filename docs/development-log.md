### 主な設計判断と根拠

#### 1. 負荷係数に `max(0, P_eq)` を使用する

当初 s(n) = P_eq^4 / v と定義したが、下り坂では P_eq が負値になるため |P_eq|^4 が大きくなり、NP制約の計算が崩壊した（急勾配の下り坂の「コスト」が平地を大幅に超える誤った結果になる）。

Coggan の NP モデルはサイクリストが実際に出力するペダルパワー（非負）の4乗平均を用いるため、`P_eff = max(0, P_eq)` を採用した。
結果として、惰性走行可能な区間（P_eq < 0）の NFD 係数は 0 になる。これは「惰性走行区間は体への負荷がない」という物理的な実態と整合する。

#### 2. 最適化アルゴリズム：一定パワー戦略

当初はラグランジュ乗数を直接求める方向で検討したが、「一定パワー戦略が NP を固定したときの時間最小化の最適解である」という古典的なペーシング理論を利用し、目標パワー P_target に対して1次元の二分探索に帰着させた。64ステップの二分探索で NP 制約の相対残差を < 1% に収束させている。

#### 3. 標準コース分布の暫定実装

実際の道路データ（OSM + 国土地理院DEM）から勾配分布を統計的に求めるツールがまだ未実装であるため、ラプラス分布的な重み `exp(-|n|/4)` で -15%〜+15% の範囲をカバーする暫定分布を使用した。この分布は将来のデータ駆動の分布で置き換えること。

#### 4. 速度の上下限クリップ

下限速度（4 km/h）は激坂での最低速度（転倒防止）として設定した。上限速度は初級 30 km/h、中級 35 km/h、上級 40 km/h とした。上級ほど高速下り坂に慣れているという経験則に基づく（問題定義の記述に準拠）。

## 2026-03-18: プリセットパラメータの修正（PRレビュー対応）

### 変更内容

`src/presets.ts` に対するレビューコメント3件を解決した。

1. **体重の修正**（body mass 70 kg → 60 kg）
   - 日本人男性平均約 67 kg、女性平均約 53 kg の中間値として 60 kg を採用。
   - 自転車質量 8 kg を加えた合計 68 kg を物理計算用の `mass` として使用。

2. **FTP 計算に体重のみを使用**（`W/kg × 体重` に修正）
   - PWR（パワーウェイトレシオ）は体重に対して定義されるため、FTP = W/kg × body_mass に修正。
   - `BODY_MASS` と `BIKE_MASS` を定数として分離し、意図を明確化。
   - 結果として FTP: 初級 120 W、中級 180 W、上級 240 W。

3. **vMax を全レベル共通 40 km/h に統一**
   - 初級・中級・上級すべて `vMax = 40 km/h`。
   - 初級でも十分に下り坂を流せることを踏まえた判断（レビューアーの指摘通り）。

## 2026-03-30: LUT参照経路の最適化とドキュメント更新

### 実施内容

勾配LUTの参照・補間を等差ビン前提で高速化し、UI側の係数参照責務をコア関数へ統一した。

1. `src/optimization/assumed-speed.ts`
   - `validateArithmeticGradeBins()` を追加。
   - 勾配ビンが「有限値」「厳密昇順」「等差数列」であることを検証。
   - `computeAssumedSpeeds()` の先頭で検証を実行。

2. `src/nfd/calculator.ts`
   - `interpolateCoefficient()` を改修。
   - LUTごとにキーグリッドを解析してキャッシュし、等差キー時は添字計算で補間（O(1)）。
   - 非等差キーは互換性のため走査ベースへフォールバック。

3. `src/optimization/lut.ts`
   - `findClosestGradeBin()` を改修。
   - 等差ビン時は丸めインデックスで最近傍ビンを求める（O(1)）。
   - `computeCourseTime()` / `computeEstimatedNp()` で解決器をループ外で再利用。

4. `ui/grade-coeff.js`
   - 係数取得を `lut.get(grade)` から `interpolateCoefficient(grade, lut)` へ変更。
   - UI側に補間ロジックを持たず、補間仕様をコアへ一元化。

5. テスト追加
   - `src/optimization/optimization.test.ts`:
     - 等差ビン検証関数の正常系・異常系
     - `findClosestGradeBin()` の等差経路/フォールバック経路
   - `src/nfd/calculator.test.ts`:
     - 等差グリッド高速経路の回帰
     - 非等差キーでのフォールバック回帰

### 検証結果

- `npx jest src/nfd/calculator.test.ts src/optimization/optimization.test.ts --runInBand` は通過。
- `npm run build` は成功。
- `npm test -- --runInBand` は `src/course-gradient/analyzer.test.ts` 実行中の長時間化により完走確認が未了。

### 変更理由

- これまでの補間処理は呼び出しごとにキー整列と走査を伴い、コース点数が多い場合のボトルネックになっていた。
- 既定ビンは `-30.0..30.0` の `0.1` 刻みで本質的に等差であるため、この前提を実装へ明示することで計算量を改善できる。
- UI側がLUT内部の離散キー前提に依存すると責務が分散するため、補間ロジックをコアへ寄せて保守性を上げる。
