# 設計ドキュメント：開発の要件と設計の根拠

## 概要

本プロジェクトは、サイクリングコースの難易度を表す加法的な指標「**平地換算距離 (Normalized Flat Distance, NFD)**」を開発・実装する。

NFD は以下の特性を持つ：

- **加法性**：コースを分割した各区間のNFDの合計が、コース全体のNFDに等しい。
- **算出が簡便**：勾配プロファイルが与えられれば、LUT（ルックアップテーブル）を引くだけで計算できる。
- **想定サイクリストごとに定義**：同じコースでも、走力（FTP）によって難易度は異なる。

---

## 要件

### 機能要件

1. **コースの勾配プロファイルからNFDを計算できること**
   - 入力：コースを短いセクションに分割した（距離, 勾配）ペアの配列
   - 出力：NFD（メートル単位またはキロメートル単位）

2. **複数の「想定サイクリスト」レベルを定義すること**
   - 初級 (beginner)：FTP 約 2.0 W/kg
   - 中級 (intermediate)：FTP 約 3.0 W/kg
   - 上級 (advanced)：FTP 約 4.0 W/kg

3. **NFD係数のLUTを勾配ごとに計算できること**
   - 0%の係数は 1.00（定義）
   - 登坂区間の係数は 1 より大きく、急な下り（惰性走行）の係数は 0

## 物理モデル

### 均衡出力（Equilibrium Power）

勾配 n%の区間を速度 v (m/s) で定速走行する際に必要な出力 P_eq (W) は、Martin et al. (1998) のモデルに基づく：

```
P_eq(v, n) = [F_gravity(n) + F_rolling(n) + F_aero(v)] × v / η
```

ただし：
- `F_gravity = m × g × sin(arctan(n/100))`（重力による抵抗/助力）
- `F_rolling = C_rr × m × g × cos(arctan(n/100))`（転がり抵抗）
- `F_aero = 0.5 × ρ × C_dA × v²`（空気抵抗）
- `η`：ドライブトレイン効率

「均衡出力」はサイクリストが実際に出力するペダル出力を意味する。**下り坂など重力の助力が大きい区間では、均衡出力は負（惰性走行で足を止められる状態）になる。この場合、サイクリストの実効出力 P_eff = max(0, P_eq) = 0 とする。**

### 速度のクリップ

サイクリストの走行速度には現実的な上下限を設ける：
- **下限 v_min ≈ 4 km/h**：激坂で転倒しないための最低速度
- **上限 v_max**（レベルごとに設定）：下り坂での惰性走行上限速度
  - 初級：30 km/h
  - 中級：35 km/h
  - 上級：40 km/h

## 負荷係数とNFD係数

### 負荷係数 s(n)

勾配 n% の区間を「想定速度」v(n) で走行した場合の負荷係数：

```
s(n) = P_eff(v(n), n)⁴ / v(n)   [W⁴·s/m]
```

`P^4` を使う根拠は Coggan の Normalized Power (NP) の定義による。
NP = (出力の30秒移動平均の4乗平均)^(1/4) は、一定ペースの走行と間欠的走行の体感的きつさを比較可能にする指標である。

### NFD係数 c(n)

```
c(n) = s(n) / s(0)
```

0%（平地）の係数を 1 に正規化する。コースのある区間の距離 d、勾配 n の貢献は `d × c(n)` となる。

---

## 想定速度の最適化

「想定速度」は、以下の最適化問題の解として定義される：

**標準コース（各勾配が既定の割合で出現するコース）全体を走行したとき、**
1. Normalized Power が「巡航パワー P_c = FTP × 0.65」に等しくなる制約のもと、
2. 総走行時間が最短になる速度プロファイル v(n) を求める。

数学的には、ラグランジュ乗数法により次の条件を満たす「目標パワー P_target」を二分探索で求める：

```
Σ_n  f(n) × [P_eff(v(n), n)⁴ − P_c⁴] / v(n) = 0
```

ただし：
- `f(n)`：標準コースにおける勾配 n% の出現頻度
- `v(n) = speedForPower(P_target, n, params)`（vMin・vMax でクリップ済み）

最適解において、速度クリップされない区間では一定パワー P_target での走行が最適となる（古典的なペーシング理論）。

## 標準コースの勾配分布（暫定）

現状では、ラプラス分布的な重み `exp(-|n|/4)` で -15% 〜 +15% の勾配に頻度を割り当てた暫定分布を使用している。
