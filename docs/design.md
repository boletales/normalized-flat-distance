# 設計ドキュメント：開発の要件と設計の根拠

## 概要

本プロジェクトは、サイクリングコースの難易度を表す加法的な指標「**平地換算距離 (Normalized Flat Distance, NFD)**」を開発・実装する。

NFD は以下の特性を持つ：

- **加法性**：コースを分割した各区間のNFDの合計が、コース全体のNFDに等しい。
- **算出が簡便**：勾配プロファイルが与えられれば、LUT（ルックアップテーブル）を引くだけで計算できる。
- **想定サイクリストごとに定義**：同じコースでも、平地基準出力 $P_0$ によって難易度は異なる。

---

## 要件

### 機能要件

1. **コースの勾配プロファイルからNFDを計算できること**
   - 入力：コースを短いセクションに分割した（距離, 勾配）ペアの配列
   - 出力：NFD（メートル単位またはキロメートル単位）

2. **複数の「想定サイクリスト」レベルを定義すること**
   - 初級 (beginner)：$P_0 = 100$ W
   - 中級 (intermediate)：$P_0 = 150$ W
   - 上級 (advanced)：$P_0 = 200$ W

3. **NFD係数のLUTを勾配ごとに計算できること**

## 物理モデル

### 均衡出力（Equilibrium Power）

勾配 n%の区間を速度 v (m/s) で定速走行する際に必要な出力 P_eq (W) は、Martin et al. (1998) のモデルに基づく：

```text
P_eq(v, n) = [F_gravity(n) + F_rolling(n) + F_aero(v)] × v / η
```

ただし：

- `F_gravity = m × g × sin(arctan(n/100))`（重力による抵抗/助力）
- `F_rolling = C_rr × m × g × cos(arctan(n/100))`（転がり抵抗）
- `F_aero = 0.5 × ρ × C_dA × v²`（空気抵抗）
- `η`：ドライブトレイン効率

これ以降、速度の三乗にかかる項を定数として `a = 0.5ρC_dA/η` と、一乗にかかる項を勾配の関数として `b(n) = mg(sin θ + C_rr cos θ)/η` と呼ぶことにする。`P_eq(v, n) = a v^3 + b(n) v` である。

「均衡出力」はサイクリストが実際に出力するペダル出力を意味する。下り坂など重力の助力が大きい区間では、均衡出力は負（惰性走行で足を止められる状態）になり得るが、この場合、サイクリストの実効出力 P_eff = max(0, P_eq) = 0 とする。

### 速度のクリップ

サイクリストの走行速度には現実的な下限を設ける：

- **下限 v_min ≈ 4 km/h**：激坂で転倒しないための最低速度

## 負荷係数とNFD係数
### 「つらさ」
コースの合計つらさを、各時間での出力 P について P^4 の時間積分として定義する。これは、各地点での出力 P について、P^4 / v を長さで積分したものと同じであるが、後者を「地点つらさ」「区間つらさ」と呼ぶことにする。

## 想定速度
「最適速度」は、あるxについて「NPがxで最速の走行」という制約の最適化問題の解となり、かつ「0%勾配での均衡出力がP_0を満たす」という条件を満たす速度関数 v(n) である。最適化問題の解の性質として、速度関数はある定数cについて各点で次の方程式を満たす（この式の左辺は端的には「区間タイムを変えたときの P^4/v の変化率」である。数理的考察については ./NFD.md を参照せよ）：

```text
(11 a v(n)² + 3 b(n)) (a v(n)² + b(n))³ v(n)⁴ = c
```

「想定速度」は、「最適速度」を v_min でクリップしたものである。

### 勾配負荷 s(n)

勾配 n% の区間を「想定速度」v(n) で走行した場合の地点のつらさである：

```text
s(n) = P_eff(v(n), n)⁴ / v(n)   [W⁴·s/m]
```

ここでは、 Coggan の Normalized Power® (NP) のもととなる四乗則を参考に、運動のストレスが出力の4乗に比例すると仮定する。

### 勾配係数 c(n)

```text
c(n) = s(n) / s(0)
```

0%（平地）の係数を 1 として正規化する。コースのある区間の距離 d、勾配 n の貢献は `d × c(n)` となる。

### 平地換算距離 NFD
コース全体にわたって c(n) を距離で積分したものである。実用に際して、コースは短い距離ごとに離散化して考え、勾配は適宜平滑化することが望ましい。

### NFDを求める手順
最初に想定速度・勾配係数を求めて LUT を構築し、そののち与えられたコースの各区間の勾配に対して LUT を引いて係数を得て、距離で積分してNFDを得る。

想定速度・勾配係数を求める手順は次のとおりである：
1. 平地速度 v_0 は `a v_0^3 + b(0) v_0 = P_0` の正の実数解であるので、数値解する。a, b(0) はいずれも正であるから、これを満たす v_0 は一意に存在し、正である。
2. 最適コスト `c := (11 a v_0^2 + 3 b(0)) (a v_0^2 + b(0))^3 v_0^4` を計算する。
3. 斜度 n における最適速度 v_op(n) は v に関する方程式 `(11 a v^2 + 3 b(n)) (a v^2 + b(n))^3 v^4 = c` の最大の実数解である。n=-30.0% から n=30.0% まで、0.1% 刻みで数値解を得る。この方程式は一つ以上正の実数解を持ち、b(n)が正の時は0より大きく、b(n)が負の時は sqrt(-b(n)/a) より大きい。
4. 斜度 n における想定速度 v_est(n) := max(v_op(n), v_min) を計算する。
5. 勾配 n における想定パワー P_est(n) := P_eff(v_est(n), n) を計算する。
6. 勾配 n における地点のつらさ s(n) := P_est(n)^4 / v_est(n) を計算する。
7. 勾配 n における勾配係数 c(n) := s(n) / s(0) を計算する。

このようにして得られた LUT を用いて、コースの各区間の勾配に対して c(n) を引いて係数を得て、距離で積分してNFDを得る。

## コース勾配の計算について

現行実装では、コース勾配は「ウェイポイント列 + 標高プロバイダ」から次の手順で算出する。

### 入力

- ウェイポイント列（緯度・経度）。
- 各点の標高（`ElevationProvider` で取得）。
- 点ごとの `interpolateElevation` フラグ（トンネル・橋梁などの補間指定）。

### 計算手順

1. ウェイポイント間の水平距離を Haversine で計算し、始点からの累積距離を作る。
2. 標高プロバイダから各点の標高（DEM生値）を取得する。
3. `interpolateElevation = true` の連続区間は、前後の非マスク点を結ぶ線形補間で埋める。
4. 始点からの距離が等間隔（既定 25m）になるように、補間後標高を線形補間してリサンプリングする。終端は元の点を消す。
5. 等間隔化した標高列に Savitzky-Golay フィルタ（既定: 窓幅 11, 多項式次数 3）を適用する。
6. 平滑化結果を元のウェイポイント位置へ再サンプリングして、勾配計算に用いる標高を得る。
7. 隣接点間の勾配を

```text
grade(%) = (elevation[i] - elevation[i-1]) / distance[i] * 100
```

で計算する（`distance[i] <= 0` はスキップ）。
8. 勾配は外れ値抑制のため `[-maxAbsGrade, +maxAbsGrade]` にクリップする（既定 ±30%）。
9. クリップ後の勾配で LUT の係数を補間し、区間距離との積を合算して NFD を得る。

### 実装上の境界条件

- ウェイポイントが空なら空プロファイルを返す。
- 補間に必要な前後端点がないマスク区間は補間しない。
- Savitzky-Golay は入力点数が不足する場合、元系列のコピーを返す。
- LUT の範囲外勾配は端の係数にピン留めされる。

### 現状の出力

- 各点について、`rawElevation`（補間後生標高）、`demElevation`（DEM生値）、`elevation`（平滑化・調整後標高）、`distanceFromStart`、`elevationSource`（`dem` または `interpolated`）を保持する。
- NFD は m と km の両方で返す。
